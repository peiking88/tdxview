# Plan: tdxview 数据可视化平台实施计划

> Source PRD: docs/PRD_TDXVIEW_VISUALIZATION.md (37个用户故事)

## 架构决策

持久的决策，适用于所有阶段：

### 技术栈
- **前端框架**: Streamlit (Python全栈方案)
- **数据存储**: DuckDB (分析型数据库) + Parquet (列式存储)
- **缓存系统**: 多级缓存 (内存 + 磁盘)
- **可视化库**: Plotly (Streamlit集成)
- **配置管理**: Pydantic + YAML配置文件 + 环境变量
- **认证授权**: JWT令牌 + 基于角色的权限控制

### 架构分层
1. **用户界面层**: Streamlit组件 (仪表板、图表、指标、配置、认证)
2. **业务逻辑层**: Python服务 (数据服务、指标服务、可视化服务、配置服务、用户服务)
3. **数据访问层**: DuckDB管理器 + Parquet管理器 + 缓存管理器
4. **数据源层**: tdxdata API适配器 + 数据源抽象接口

### 数据模型
- **users表**: 用户认证、配置和权限
- **data_sources表**: tdxdata配置和多数据源支持
- **indicators表**: 技术指标定义和参数配置
- **dashboards表**: 用户仪表板布局和小部件配置
- **time_index表**: 时间分区索引
- **asset_index表**: 资产代码索引

### API设计原则
- 使用Pydantic模型进行请求/响应验证
- RESTful风格API端点
- 统一的错误处理机制
- 版本化API支持 (预留)

### 部署架构
- 容器化部署 (Docker + Docker Compose)
- 环境变量配置管理
- 健康检查端点 (/health)
- 结构化日志输出 (JSON格式)
- 监控集成 (Prometheus + Grafana)
